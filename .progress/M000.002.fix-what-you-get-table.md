# M0 — Fix broken "What you get" table in README

## Summary

The table under the "## What you get" heading in the root [README.md](../README.md) was broken due to Markdown parsing. The second row used an unescaped `*` character inside the link text: `[.progress/M000.*.md](.progress/)`. Asterisks in link text are interpreted as emphasis markers by Markdown, causing GitHub to render the link label incorrectly (showing a literal backslash escape like `M000.\*.md` in the output). Fixed by properly escaping the asterisk as `\*` in the source link text.

## Changes

- Fixed link text in [README.md](../README.md) "What you get" table row 2: changed `[.progress/M000.*.md]` to `[.progress/M000.\*.md]`
- Created this progress entry to record the fix per the immutable append-only convention.

## Decisions

- Use standard Markdown escaping (`\*`) for literal asterisks inside link labels and other inline formatting contexts.
- Adhere strictly to the progress log rules: only append new files, never modify existing committed entries (including this one later).
- This fix is part of completing M0 Bootstrap "README explains what Dot Progress is".
