# General - Clarify bootstrap and conflict rules

## Summary

Updated the project README to make cross-repository adoption more deterministic when users provide only a repo URL and ask an agent to implement Dot Progress. The new guidance adds a copy/paste bootstrap prompt and explicit precedence rules so agents do not misinterpret when to append entries.

## Changes

- Updated `README.md` with a one-shot setup flow for applying Dot Progress in other repositories
- Added a reusable bootstrap prompt that enforces append-only behavior and strict naming
- Clarified that `.progress/README.md` is authoritative when instructions conflict
- Tightened onboarding text to include partial-progress and decision logging expectations

## Decisions

- Keep `.progress/README.md` as the single source of truth for behavior and format
- Document common failure modes in the root README so users can prevent ambiguity before agent execution

## Follow-ups

- Consider adding a minimal `AGENTS.md` template snippet directly in `README.md` for copy/paste onboarding
