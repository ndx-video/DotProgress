# General - Enforce OKF type minimum

## Summary

Adjusted Dot Progress documentation and examples to enforce OKF minimum compliance by requiring `research.type` whenever a `research` block is present in frontmatter.

## Changes

- Updated `.progress/README.md` template and metadata section to require `research.type` for OKF compliance
- Updated `.claude/skills/dotprogress/SKILL.md` workflow guidance to include the `research.type` minimum
- Updated `.progress/general/G000003.example-research-metadata-entry.md` to include `research.type: progress_entry`
- Updated root `README.md` to mention the `research.type` minimum

## Decisions

- Keep core required Dot Progress fields unchanged (`file`, `date`, `milestone`, `related_commits`)
- Require `research.type` only when `research` metadata is present, preserving backward compatibility for entries without `research`

## Follow-ups

- Define an organization-wide controlled vocabulary for `research.type` values if multiple OKF type labels are expected
