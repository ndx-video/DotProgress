# M0 — Subfolder layout + agent-harness infrastructure

## Summary

Built on the prior commits that added General/Fix prefixes: entries are now grouped
into **type subfolders** (`milestone/`, `general/`, `fixes/`) for human browsability,
and the repo now ships an optional **agent-harness layer** (`.claude/rules` +
`.claude/skills`) so AI agents enforce the convention automatically. This is a
one-time, maintainer-led structural migration of the log itself, recorded here per
the immutability rule.

## Changes

- `.progress/` reorganized into `milestone/`, `general/`, `fixes/`. The existing
  example entries (`M000.001`, `G000001`, `F000001`) were moved into their subfolders
  and their relative links fixed.
- `.progress/README.md` updated: directory-layout section, subfolder-aware filename
  rules / finding-next commands, and an "Agent integration" section.
- Added agent-harness infrastructure (generic, copy-paste portable):
  `.claude/rules/dotprogress.md` (path-glob rule) and
  `.claude/skills/dotprogress/SKILL.md` (append-an-entry skill).
- Updated `README.md` ("What you get" + adoption steps), `ROADMAP.md` (working
  convention), and `AGENTS.md` (subfolders + `.claude` layer).

## Decisions

- Counters remain **global per prefix** (not per subfolder); subfolders are purely
  for human browsability.
- A maintainer-led structural migration of the log is the sole sanctioned exception
  to immutability, and must be documented as a new entry (this file).

## Follow-ups

- Adopters copy `.progress/` + (optionally) `.claude/` as-is; no project-specific edits.
